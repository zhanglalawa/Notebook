## 观察者模式
定义了对象之间的一对多依赖（一个主题对应多个观察者），这样一来，当一个对象改变状态时候，它的所有依赖者都会受到通知并且自动更新。

## push形式的观察者模式
我们这里实现的例子是被观察者信息改变之后，由主题（被观察者）把信息push出去，通知观察者，观察者做出反应。

## pull形式的观察者模式
JDK中也有自带的观察者模式。使用Observalble来实现的。但是被观察者是一个类而不是接口，限制了它的复用能力。另外Observalbal实现了notifyObservers，具体怎么实现的不用管了。。都废弃了，这就带来了麻烦，导致通知观察者的次序和我们先前的依次添加的次序是不一样的，可能会以一个我们不希望的顺序通知。

它和我们上面实现的观察者模式不同之处在于它是给主题定义了一系列getter，并没有在update中传送这些数据，只是传过去了主题对象，而是由观察者在update方法中通过这个对象的getter来获取改变的数据的，我们称之为pull的方法。观察者比较主动。

鉴于JDK9中已经弃用了这个类，也存在很多问题，所以就先放弃不写例子了。
